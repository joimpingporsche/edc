Definition: Interface In the context of enterprise architecture, an Interface represents a logical connection between two IT Products. It models the way in which data is exchanged between these IT Products, including the direction, frequency, and type of information being transferred.
Purpose and benefits The Interface object in LeanIX serves as a critical link between the Application Architecture and the Information & Data Architecture. It provides structured metadata about:
• What type of information objects are exchanged (e.g., master data, transactional data) • How the exchange occurs (e.g., synchronous, asynchronous) • In which direction the data flows (e.g., unidirectional, bidirectional) • At what frequency the exchange takes place (e.g., real-time, daily, batch) By modeling interfaces, architects gain visibility into data dependencies, integration complexity, and potential risks across the application landscape. This supports:
• Improved data governance • Better integration design • Enhanced impact analysis • Alignment with SAFe value streams and IT product boundaries Structure Summary: Interface Modeling in LeanIX Responsible / Subscriptions Each Interface Fact Sheet requires a person in the subscriptions who is responsible for the content and quality of the data. The assignment of type, person and role is done via the subscriptions in the respective fact sheet
